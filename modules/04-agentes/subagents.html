Subagentes & Agent Teams

Padrão de uso onde múltiplos agentes especializados trabalham em paralelo, cada um com contexto limpo, coordenados por um orquestrador.

O que são Agent Teams

Agent Teams é um padrão arquitetural onde múltiplos agentes especializados trabalham em paralelo, cada um com contexto limpo (~50–80k tokens), coordenados por um orquestrador na janela principal.

Padrão orquestrador

A estrutura hierárquica de um agent team:

Orquestrador (janela principal)
├── Mantém estado global (só paths, não conteúdo)
├── Spawna agentes em waves
├── Coleta resultados (só { status, file })
└── Faz merge/integração ao final

Agentes (janelas separadas, ~50-80k tokens)
├── Recebem brief compacto
├── Leem context de arquivos
├── Executam tarefa específica
└── Escrevem output em arquivo
   └── Retornam só { status, file }

Exemplo real — paralelizando análise de codebase

Um caso prático de agent team para análise paralela de um projeto:

Agente 1: analisa src/components/ → escreve analysis-components.json
Agente 2: analisa src/api/        → escreve analysis-api.json
Agente 3: analisa src/lib/        → escreve analysis-lib.json
Agente 4: analisa tests/          → escreve analysis-tests.json
Agente 5: analisa docs/           → escreve analysis-docs.json

Orquestrador: lê os 5 arquivos JSON → gera relatório final

Regras para agent teams eficientes

Máximo 5 agentes paralelos (evita rate limiting)
Cada agente < 100k tokens de contexto
Retorno ao orquestrador: sempre só paths/status
Agentes transientes: não dependem de sessão persistente
Handoff via arquivos: output de um → input do próximo

Passagem de bastão (handoff) entre agentes

Agentes podem trabalhar em sequência (waves), onde o output de um se torna o input do próximo:

Wave 1: A1 → escreve design.json
Wave 2: B1 lê design.json → produz component.html
Wave 3: C1 lê component.html → escreve tests

Anti-padrões

Erros comuns a evitar ao usar agent teams:

❌ Agente retorna conteúdo completo ao orquestrador
❌ Agente com brief de 10.000 tokens
❌ 20 agentes em paralelo
❌ Agente lê tudo do projeto para fazer tarefa simples
❌ Orquestrador processa HTML/JSON completo inline

Exercício

Crie um prompt que instrua o Claude a usar 3 agentes paralelos para: (1) pesquisar melhores práticas de TypeScript, (2) pesquisar melhores práticas de React, (3) pesquisar melhores práticas de testes. Cada agente deve escrever seu resultado em um arquivo separado (ex: best-practices-typescript.md). O orquestrador só recebe os paths de volta — nunca o conteúdo completo. Por fim, consolide em um arquivo final.
