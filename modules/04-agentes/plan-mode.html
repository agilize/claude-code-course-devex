Plan Mode

Plan Mode é um modo onde Claude planeja o que fará sem executar. Útil para revisar o plano antes de permitir execução.

O que é Plan Mode

Plan Mode é um modo onde Claude detalha todas as ações que tomaria para completar uma tarefa — sem executar nenhuma delas. Você revisa o plano e decide se aprova ou modifica antes de qualquer ação real.

Ativando Plan Mode

Você pode ativar o Plan Mode via prompt natural:

# Via prompt direto
"Planeje (não execute) como você refatoraria este módulo"
"Entre em plan mode: como você implementaria autenticação JWT?"

# O Claude também usa plan mode automaticamente em tarefas de alto risco

Como funciona

O fluxo do Plan Mode em 4 passos:

  1. Claude analisa a tarefa
  2. Lista todas as ferramentas que usaria e por quê
  3. Mostra o impacto esperado (arquivos modificados, comandos executados)
  4. Aguarda aprovação antes de executar qualquer ação

Quando usar Plan Mode

Situações onde Plan Mode é especialmente valioso:

  • Antes de refatorações grandes
  • Antes de deletar ou mover arquivos
  • Em novos projetos (entender o que Claude fará)
  • Para aprender como Claude abordaria um problema
  • Em produção (validar antes de executar)

ExitPlanMode

Quando Claude está em Plan Mode, você pode aprovar o plano como está, pedir modificações, ou cancelar. Ao aprovar, o Claude executa exatamente o que foi planejado.

Exemplo de interação em Plan Mode

Como se parece uma sessão típica em Plan Mode:

Você: "Planeje como você migraria este projeto de JavaScript para TypeScript"

Claude (plan mode):
1. Analisarei a estrutura atual com Glob('**/*.js')
2. Identificarei as dependências com Read(package.json)
3. Para cada arquivo .js:
   a. Renomeio para .ts
   b. Adiciono tipos explícitos
   c. Corrijo erros de compilação
4. Atualizarei tsconfig.json
5. Rodando npm run build para verificar

Deseja prosseguir? [Aprovar / Modificar]

Exercício

Escolha um projeto seu e peça ao Claude: "Planeje (não execute) como você migraria este projeto para usar ESModules". Analise o plano gerado e identifique passos que você gostaria de modificar antes de aprovar a execução.
